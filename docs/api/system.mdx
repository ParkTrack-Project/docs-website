# 5. System

### 5.1 `GET /health`

Используется системой мониторинга для проверки доступности и работоспособности сервера API.

Авторизация не требуется.

**Response (200)**
```json
{"status":"ok"}
```

**Ошибки**

см. [общие ошибки](/docs/api/#общие-ошибки).

---

### 5.2 `GET /version`

Возвращает версию API.

**Response (200)**
```json
{"api":"v0.0"}
```

**Ошибки**

см. [общие ошибки](/docs/api/#общие-ошибки).
