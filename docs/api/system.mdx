# 5. System

### 5.1 `GET /health`

Используется системой мониторинга для проверки доступности и работоспособности сервера API и БД.

Авторизация не требуется.

**Response (200)**
```json
{"status":"healthy"}
```

**Response (200) - в случае отсутствия подключения к БД:**
```json
{"status":"degraded"}
```

**Ошибки**

см. [общие ошибки](/docs/api/#общие-ошибки).

---

### 5.2 `GET /version`

Возвращает версию API.


см. [общие ошибки](/docs/api/#общие-ошибки).
