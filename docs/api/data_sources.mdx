# 10. Data Sources

Раздел описывает подключённые источники данных, через которые система получает сведения о парковках, занятости, оплатах и других сигналах.

## 10.1 Назначение раздела

Раздел `Data Sources` используется для:

- регистрации источников данных партнёров;
- хранения конфигурации источников;
- управления жизненным циклом источников;
- отображения статуса подключения и последней синхронизации;
- последующего расширения системы за пределы камер.

Раздел предназначен для будущих интеграций и для административного/партнёрского интерфейса.

> Камеры продолжают описываться в разделе `Cameras` и не заменяются этим разделом.  
> В MVP камера является отдельной сущностью, а не обязательной записью в `Data Sources`.

---

## 10.2 Совместимость с текущей реализацией

Раздел `Data Sources` является **новым** и не заменяет существующие эндпоинты `Cameras`.

Для сохранения совместимости:

- текущий код, работающий только с камерами, может не использовать `Data Sources`;
- новые интеграции могут регистрироваться через `Data Sources` без изменения API камер;
- при необходимости камера и источник данных могут быть связаны на уровне реализации через `partner_id` и служебные ссылки в `metadata`.

## 10.3 Общие правила

### Авторизация

Все эндпоинты раздела требуют авторизации.

```http
Authorization: Bearer <access_token>
```

### Модель доступа

Для эндпоинтов раздела используются следующие разрешения:

- `sources.view` — просмотр источников данных;
- `sources.create` — создание источников данных;
- `sources.update` — изменение источников данных;
- `sources.delete` — удаление источников данных;
- `admin.partners.manage` — полный доступ ко всем источникам данных;
- `admin.system.view` — просмотр состояния интеграций в административных целях.

### Правила видимости данных

- администратор платформы может просматривать и изменять все источники данных;
- сотрудник партнёра может работать только с источниками своей организации;
- область доступа к данным (`read_scope`, `write_scope`, `delete_scope`) применяется к источникам так же, как и к другим ресурсам партнёра.

### Общие ошибки

| Код     | Тип                      | Описание                                                                 |
|---------|--------------------------|--------------------------------------------------------------------------|
| **400** | `Bad Request`            | Невалидный JSON или неверные типы полей.                                 |
| **401** | `Unauthorized`           | Токен отсутствует, невалиден, истёк или сессия завершена.                |
| **403** | `Forbidden`              | У пользователя недостаточно прав для выполнения действия.                |
| **404** | `Not Found`              | Источник данных или партнёр не найдены.                                  |
| **409** | `Conflict`               | Конфликт данных.                                                         |
| **415** | `Unsupported Media Type` | Неверный `Content-Type`. Используй `application/json`.                   |
| **422** | `Unprocessable Entity`   | Ошибка валидации.                                                        |
| **500** | `Internal Server Error`  | Необработанная ошибка сервера.                                           |
| **503** | `Service Unavailable`    | Сервис временно недоступен.                                              |

Пример ответа:

```json
{
  "error_description": "Validation error: source_type is required"
}
```

## 10.4 Формат данных

### Время

Все временные значения передаются в формате UTC ISO 8601:

```text
2026-04-06T10:00:00Z
```

### Тип источника

Поле `source_type` определяет тип подключённого источника данных.

Рекомендуемые значения:

- `camera_stream` — видеопоток камеры;
- `partner_api` — API партнёра;
- `parking_sensor` — парктроники и датчики занятости;
- `payment_feed` — данные об оплатах;
- `navigator_feed` — агрегированные навигационные данные;
- `carsharing_feed` — данные каршеринга;
- `manual_import` — ручной импорт или загрузка файла;
- `other` — иной источник.

> В MVP фактически могут использоваться только будущие интеграции или тестовые записи.  
> Текущая работа системы через раздел `Cameras` сохраняется.

### Статус источника

Поле `status` описывает текущее состояние источника.

Поддерживаемые значения:

- `draft` — источник создан, но ещё не активирован;
- `active` — источник активен;
- `paused` — источник временно отключён;
- `error` — источник в ошибочном состоянии;
- `deleted` — источник деактивирован.

### Конфигурация источника

Поле `config` хранит параметры подключения и настройки интеграции в формате JSON.

Поле:

- может содержать URL, ключи, идентификаторы, интервалы опроса и другие настройки;
- в ответах API чувствительные значения могут быть скрыты или замаскированы;
- структура `config` зависит от `source_type`.

## 10.5 Модель `DataSource`

Полная модель источника данных.

- `source_id` (`integer`) — уникальный идентификатор источника.
- `partner_id` (`integer`) — идентификатор партнёра-владельца.
- `created_by_user_id` (`integer | null`) — пользователь, создавший источник.
- `name` (`string`) — название источника.
- `source_type` (`string`) — тип источника данных.
- `status` (`string`) — статус источника.
- `description` (`string | null`) — описание источника.
- `config` (`json`) — конфигурация источника.
- `last_sync_at` (`string | null`, ISO 8601) — время последней успешной синхронизации.
- `last_error` (`string | null`) — последнее сообщение об ошибке.
- `is_active` (`boolean`) — активен ли источник.
- `created_at` (`string`, ISO 8601) — дата создания.
- `updated_at` (`string`, ISO 8601) — дата последнего обновления.

**Пример**

```json
{
  "source_id": 3001,
  "partner_id": 10,
  "created_by_user_id": 123,
  "name": "API платной парковки центра",
  "source_type": "partner_api",
  "status": "active",
  "description": "Интеграция с API поставщика данных о парковках.",
  "config": {
    "base_url": "https://partner.example/api",
    "poll_interval_seconds": 60,
    "auth_type": "api_key",
    "api_key_masked": "****abcd"
  },
  "last_sync_at": "2026-04-06T10:40:00Z",
  "last_error": null,
  "is_active": true,
  "created_at": "2026-04-06T09:00:00Z",
  "updated_at": "2026-04-06T10:40:00Z"
}
```

## 10.7 `GET /sources`

Возвращает список источников данных, доступных текущему пользователю.

### Правила доступа
- администратор платформы может видеть все источники;
- сотрудник партнёра может видеть только источники своей организации;
- фильтрация по `partner_id` допускается только в пределах доступных данных.

**Требуемые разрешения**
- `sources.view`

**Headers**
- `Authorization: Bearer <access_token>`

**Query-параметры (необязательные)**

- `partner_id` (`integer`) — фильтр по партнёру-владельцу.
- `source_type` (`string`) — фильтр по типу источника.
- `status` (`string`) — фильтр по статусу.
- `is_active` (`boolean`) — фильтр по активности.
- `q` (`string`) — поиск по `name` и `description`.
- `top` (`integer`, 1..100) — максимальное число элементов.
- `offset` (`integer`, >=0) — смещение для пагинации.
- `view` (`string`) — режим ответа. Поддерживаемые значения:
  - `list` — облегчённый список (по умолчанию);
  - `full` — полная модель.

**Пример запроса**

```http
GET /api/v1/sources?partner_id=10&status=active&view=list&top=20&offset=0 HTTP/1.1
Host: api.parktrack.live
Authorization: Bearer <token>
```

**Response (200)**

- при `view=list`:
  - `items` (`DataSourceListItem[]`)
  - `total` (`integer`)
  - `top` (`integer`)
  - `offset` (`integer`)
- при `view=full`:
  - `items` (`DataSource[]`)
  - `total` (`integer`)
  - `top` (`integer`)
  - `offset` (`integer`)

**Пример ответа (200) — `view=list`**

```json
{
  "items": [
    {
      "source_id": 3001,
      "partner_id": 10,
      "name": "API платной парковки центра",
      "source_type": "partner_api",
      "status": "active",
      "last_sync_at": "2026-04-06T10:40:00Z",
      "last_error": null,
      "is_active": true
    }
  ],
  "total": 1,
  "top": 20,
  "offset": 0
}
```

**Пример ответа (200) — `view=full`**

```json
{
  "items": [
    {
      "source_id": 3001,
      "partner_id": 10,
      "created_by_user_id": 123,
      "name": "API платной парковки центра",
      "source_type": "partner_api",
      "status": "active",
      "description": "Интеграция с API поставщика данных о парковках.",
      "config": {
        "base_url": "https://partner.example/api",
        "poll_interval_seconds": 60,
        "auth_type": "api_key",
        "api_key_masked": "****abcd"
      },
      "last_sync_at": "2026-04-06T10:40:00Z",
      "last_error": null,
      "is_active": true,
      "created_at": "2026-04-06T09:00:00Z",
      "updated_at": "2026-04-06T10:40:00Z"
    }
  ],
  "total": 1,
  "top": 20,
  "offset": 0
}
```

**Ошибки**

| Код     | Тип             | Описание                                   |
|---------|-----------------|--------------------------------------------|
| **401** | `Unauthorized`  | Токен отсутствует, невалиден или истёк.    |
| **403** | `Forbidden`     | Недостаточно прав для просмотра источников. |

## 10.8 `POST /sources/new`

Создаёт новый источник данных.

**Требуемые разрешения**
- `sources.create`

**Требуемая область доступа**
- `write_scope` должен разрешать создание источника в выбранной области данных.

**Headers**
- `Authorization: Bearer <access_token>`
- `Content-Type: application/json`

**Request body (required)**

- `partner_id` (`integer`) — идентификатор партнёра-владельца.
- `name` (`string`, 1..200) — название источника.
- `source_type` (`string`) — тип источника.
- `config` (`json`) — конфигурация источника.

**Request body (optional)**

- `description` (`string`) — описание источника.
- `status` (`string`) — начальный статус. По умолчанию: `draft`.
- `is_active` (`boolean`) — активность источника. По умолчанию: `true`.

**Пример запроса**

```http
POST /api/v1/sources/new HTTP/1.1
Host: api.parktrack.live
Authorization: Bearer <token>
Content-Type: application/json

{
  "partner_id": 10,
  "name": "API платной парковки центра",
  "source_type": "partner_api",
  "description": "Интеграция с API поставщика данных о парковках.",
  "config": {
    "base_url": "https://partner.example/api",
    "auth_type": "api_key",
    "api_key": "secret_value",
    "poll_interval_seconds": 60
  }
}
```

**Response (201)**

- `source_id` (`integer`) — идентификатор созданного источника.

**Пример ответа (201)**

```json
{
  "source_id": 3001
}
```

**Ошибки**

| Код     | Тип                    | Описание                                                          |
|---------|------------------------|-------------------------------------------------------------------|
| **400** | `Bad Request`          | Невалидное тело запроса.                                          |
| **401** | `Unauthorized`         | Токен отсутствует, невалиден или истёк.                           |
| **403** | `Forbidden`            | Недостаточно прав для создания источника.                         |
| **404** | `Not Found`            | Указанный `partner_id` не найден.                                 |
| **409** | `Conflict`             | Источник с таким именем уже существует в рамках партнёра.         |
| **415** | `Unsupported Media Type` | Неверный `Content-Type`.                                        |
| **422** | `Unprocessable Entity` | Ошибка валидации полей источника или конфигурации.                |

## 10.10 `PUT /sources/<source_id>`

Обновляет источник данных.

Допускается частичное обновление.

**Path-параметры**
- `source_id` (`integer`, required)

**Требуемые разрешения**
- `sources.update`

**Требуемая область доступа**
- `write_scope` должен включать целевой источник данных.

**Headers**
- `Authorization: Bearer <access_token>`
- `Content-Type: application/json`

**Request body**
- `name` (`string, optional`) — название источника.
- `description` (`string | null, optional`) — описание источника.
- `status` (`string, optional`) — статус источника.
- `config` (`json, optional`) — конфигурация источника.
- `is_active` (`boolean, optional`) — активность источника.

> Поля `source_id`, `partner_id`, `created_by_user_id`, `last_sync_at`, `last_error`, `created_at`, `updated_at` изменяются сервером и игнорируются в теле запроса.

Host: api.parktrack.live
Authorization: Bearer <token>

**Response (200)** — объект `DataSource`

## 10.11 `DELETE /sources/<source_id>`

Удаляет источник данных.

В MVP удаление может быть реализовано как мягкое удаление с переводом:

- `status = deleted`
- `is_active = false`

**Path-параметры**
- `source_id` (`integer`, required)

**Требуемые разрешения**
- `sources.delete`

**Требуемая область доступа**
- `delete_scope` должен включать целевой источник данных.

**Headers**
- `Authorization: Bearer <access_token>`

**Пример запроса**

```http
DELETE /api/v1/sources/3001 HTTP/1.1
Host: api.parktrack.live
Authorization: Bearer <token>
```

**Response (204)**  
Тело ответа отсутствует.

**Ошибки**

| Код     | Тип             | Описание                                  |
|---------|-----------------|-------------------------------------------|
| **401** | `Unauthorized`  | Токен отсутствует, невалиден или истёк.   |
| **403** | `Forbidden`     | Недостаточно прав для удаления источника. |
| **404** | `Not Found`     | Источник не найден.                       |

## 10.12 Требования к другим разделам API

Раздел `Data Sources` рекомендуется использовать как справочник подключённых интеграций партнёра.

Раздел `Cameras` остаётся основным для управления видеопотоками и их геометрией.

Разделы `Occupancy` и `Forecasts` могут использовать поля:

- `source_type`
- `source_ref`
- `metadata`

для связи наблюдений и прогнозов с конкретным источником данных или внешним событием.
